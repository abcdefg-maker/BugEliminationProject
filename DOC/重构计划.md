# 波咕清除计划 — 代码重构方案

## 背景

项目是一个 Unity 2D 叙事冒险解谜游戏，共 23 个 C# 脚本。当前代码存在以下核心问题：

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| GameStateManager 上帝类 | 高 | 194行，Update() 75+ 行条件逻辑，混合标志管理/场景切换/对象定位/日程推进 |
| 魔法字符串泛滥 | 高 | 40+ 个硬编码标志名、场景名、标签名散布于 9 个文件 |
| 魔法数字泛滥 | 高 | 30+ 个硬编码速度、延迟、位置向量散布于多个文件 |
| 对话逻辑重复 | 中 | MultiDialogueTrigger 和 NPC 中完全相同的阶段选择代码 |
| 无命名空间 | 中 | 全部 23 个脚本在全局命名空间 |
| 文件名类名不匹配 | 低 | 6 个文件（如 Work.cs 内部类名是 StringSelectionHandler） |
| 未使用代码 | 低 | Entity.cd、PlayerState.stateTimer/i 等从未读取 |
| NPC 轮询对话状态 | 中 | 用 while 循环检查面板是否关闭，应改为事件 |

## 硬性约束

- **不改变任何 public 字段**（Unity Inspector 绑定）
- **不改变类名**（场景/预制体序列化引用）
- **不改变游戏行为**（重构，非功能变更）

---

## 执行步骤

### 第一步：清除未使用代码

**优先级：高 | 风险：零**

| 文件 | 移除内容 | 原因 |
|------|----------|------|
| Entity.cs | `cd` 属性 (CapsuleCollider2D) | 从未赋值，Start() 中无 GetComponent 调用 |
| Entity.cs | `ReturnDefaultSpeed()` 方法 | 无调用者，无子类覆写 |
| PlayerState.cs | `int i = 0` | 调试变量，无实际用途 |
| PlayerState.cs | `stateTimer` 字段 | 在 Update() 中递减但从未被读取或检查 |
| DialogueTrigger.cs | 已注释的旧 DialogueTrigger 类 | 应使用版本控制而非注释保留 |

---

### 第二步：提取常量

**优先级：高 | 风险：极低**

新建 `Assets/Scripts/Constants/GameConstants.cs`，包含：

```
GameFlags    — Thanks, Result, Day4, InDanger, CanLeave, Day1End, Day2Start 等 16+ 个标志名
GameScenes   — OfficeScene, ThanksScene, ResultScene, MessScene, WorkScene, PuzzleScene, MainMenu
GameTags     — Player
AnimParams   — Idle, Move, yVelocity
```

替换涉及文件：GameStateManager.cs (19处), Work.cs (6处), LeavePuzzle.cs (3处), MonsterChasing.cs (2处), WorkSceneTrigger.cs, MainMenu.cs, ThanksRollingController.cs, PlayerState.cs, Player.cs

---

### 第三步：提取魔法数字

**优先级：高 | 风险：极低**

在 GameStateManager.cs 顶部定义命名常量：

```csharp
private static readonly Vector3 BossOffScreenPos = new Vector3(17, 2, 0);
private static readonly Vector3 PlayerDoorPos = new Vector3(-6, 4, 0);
private static readonly Vector3 WangDay4Pos = new Vector3(2.7f, -1.78f, 0);
private static readonly Vector3 WangDoorPos = new Vector3(-3, 3.6f, 0);
private static readonly Vector3 PlayerSelfTalkPos = new Vector3(-1.7f, -1.5f, 0);
```

同样处理 DialogueManager.cs 中的立绘偏移和 PlayerMoveState.cs 中的 Y 轴速度系数。

---

### 第四步：分解 GameStateManager.Update()

**优先级：高 | 风险：低**

将 75+ 行 Update() 拆分为职责方法 + 一次性执行模式：

```
HandleSceneTransitionFlags()  — Thanks/Result 标志 → 场景切换
HandleDay4Setup()             — Day4 标志 → 角色位置/怪物范围
HandleDay1End()               — Day1End → 推进 Day2
HandleDay2Events()            — SelfTalkDay2/Day2End → 推进 Day3
```

关键改进：GameObject.Find() 从每帧调用改为一次性（bool 守卫）。

---

### 第五步：统一对话阶段选择

**优先级：中 | 风险：低**

新建 `Assets/Scripts/DialogueSystem/DialogueStageResolver.cs`：
- 定义 `IDialogueStage` 接口
- 静态方法 `DialogueStageResolver.Resolve()` 替代两处重复逻辑
- `DialogueStage` 和 `NPCDialogueStage` 通过显式接口实现适配（不影响序列化字段）

---

### 第六步：添加对话结束事件

**优先级：中 | 风险：低**

- DialogueManager 新增 `System.Action onDialogueEnd`
- NPC.WaitForDialogueEnd() 从轮询改为事件订阅

---

### 第七步：文件重命名

**优先级：低 | 风险：零**（Unity 通过 .meta GUID 追踪）

| 当前 | 重命名为 |
|------|----------|
| LeavePuzzle.cs | SceneTransition.cs |
| MonsterChasing.cs | ChasePlayerWithDeathTMP.cs |
| ThanksRollingController.cs | CreditsRoll.cs |
| Work.cs | StringSelectionHandler.cs |
| DayNightTransition.cs | DayNightManager.cs |
| DialogueTrigger.cs | MultiDialogueTrigger.cs |

---

### 第八步：添加命名空间

**优先级：中 | 风险：中等**

统一命名空间 `BugElimination`。需在 Unity Editor 中验证 ScriptableObject 资源反序列化。

---

### 第九步：文件夹重组

**优先级：低 | 风险：零**

```
Assets/Scripts/
├── Constants/      — 常量定义
├── DialogueSystem/ — 对话系统
├── Entity/         — 角色（含 Player/）
├── StateManager/   — 状态管理
├── Scene/          — 场景切换相关
├── Camera/         — 摄像机
├── Gameplay/       — 游戏玩法逻辑
└── Util/           — 工具类
```

---

## 不做的事情

- 不添加完整事件总线（项目规模小，星形耦合可控）
- 不重命名类名（破坏场景引用）
- 不重命名 public 字段（丢失 Inspector 绑定）
- 不移除旧对话系统（需先审计 ScriptableObject 资源）
